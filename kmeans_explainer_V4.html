K-Means Clustering

An unsupervised algorithm that partitions data into K groups by iteratively assigning points to the nearest centroid and recomputing centroids — until convergence. No labels needed.

// the k-means algorithm — step through it
Step 1: Place K centroids randomly
Step 2: Assign each point to its nearest centroid (Euclidean distance)
Step 3: Recompute centroids as the mean of assigned points
Step 4: Repeat steps 2–3 until centroids stop moving

// distance formula — the core measure
d(x, μ) = √ [ (x₁ − μ₁)² + (x₂ − μ₂)² ]
📐
Each point is assigned to the cluster whose centroid (μ) is closest. This is just Euclidean distance — the straight-line distance you learned in school. K-Means minimises the total within-cluster sum of squares (WCSS).
// how many clusters? — the elbow method

Run K-Means for K = 1 to 8. Plot the WCSS (inertia) — the total squared distance from each point to its centroid. As K increases, WCSS always decreases. The elbow — where improvement slows sharply — is a good candidate for optimal K.

💪
Elbow at K=4: adding a 5th cluster gives much less improvement than going from 3→4. The curve clearly "bends" here. This is the elbow point — a heuristic, not a rule.
// silhouette score — a better metric

The silhouette score measures how well each point fits its own cluster vs. the nearest neighbouring cluster. Score ranges from −1 (wrong cluster) to +1 (perfect fit). Higher = better separated clusters.

s(i) = ( b(i) − a(i) ) / max( a(i), b(i) )
a(i) = mean dist to own cluster  |  b(i) = mean dist to nearest other cluster
✅
K=4 scores highest on both methods. Unlike the elbow, the silhouette score has a clear maximum — making it less ambiguous. Always use both methods together.
